P09

Deep Learning

Jaringan saraf tiruan berlapis (multi-layer) yang belajar merepresentasikan data secara hierarkis, dari fitur sederhana ke fitur kompleks.

01Neuron & Multi-Layer Perceptron (MLP)

Satu neuron menghitung jumlah berbobot dari input lalu melewatkannya ke fungsi aktivasi. MLP menyusun neuron dalam input layer → hidden layer(s) → output layer, di mana tiap layer terhubung penuh (fully connected) ke layer berikutnya.

Output neuronz = Σ(w_i · x_i) + b a = f(z) → f = fungsi aktivasi

02Fungsi Aktivasi

FungsiRumusKegunaan
Sigmoidσ(x) = 1 / (1 + e^-x)Output biner (0–1), rentan vanishing gradient
Tanh(e^x − e^-x) / (e^x + e^-x)Output −1 s.d. 1, zero-centered
ReLUf(x) = max(0, x)Default hidden layer, cepat & efisien
Leaky ReLUx jika x>0, else αxMengatasi "dying ReLU"
Softmaxe^xi / Σ e^xjOutput layer klasifikasi multi-kelas
Sering ditanya: mengapa ReLU lebih disukai dari Sigmoid di hidden layer? Karena Sigmoid menyebabkan gradien mengecil drastis (vanishing gradient) saat backpropagation melewati banyak layer, sedangkan ReLU tidak menjenuh untuk nilai positif.

03Forward Propagation, Loss, & Backpropagation

Forward propagation: data mengalir dari input ke output menghasilkan prediksi ŷ. Loss function mengukur selisih ŷ dengan label asli y. Backpropagation menghitung gradien loss terhadap tiap bobot menggunakan aturan rantai (chain rule), lalu bobot diperbarui via gradient descent.

LossMSE (regresi) = (1/n) Σ (y − ŷ)² Cross-Entropy (klasifikasi) = − Σ y·log(ŷ)
Update bobot (gradient descent)w := w − η · ∂L/∂w η = learning rate

04Optimizer

  • SGD — update per batch kecil, bisa berosilasi.
  • Momentum — menambah "inersia" dari update sebelumnya agar konvergen lebih stabil.
  • RMSProp — menyesuaikan learning rate per parameter berdasarkan rata-rata kuadrat gradien.
  • Adam — gabungan Momentum + RMSProp, paling umum dipakai default saat ini.

05Regularisasi (Mencegah Overfitting)

  • L1 / L2 Regularization — menambahkan penalti pada bobot besar di loss function.
  • Dropout — mematikan sebagian neuron secara acak saat training agar model tidak bergantung pada neuron tertentu.
  • Batch Normalization — menormalkan output tiap layer agar training lebih stabil & cepat.
  • Early Stopping — menghentikan training saat validation loss mulai naik.

06CNN (Convolutional Neural Network)

Dirancang untuk data grid seperti gambar. Convolution layer menggeser filter/kernel kecil ke seluruh gambar untuk mengekstrak fitur (tepi, tekstur, pola). Pooling layer (max/average) mengurangi dimensi spasial sekaligus mempertahankan fitur penting.

Ukuran output convolutionOutput = ((W − F + 2P) / S) + 1 W=ukuran input, F=ukuran filter, P=padding, S=stride

07RNN & LSTM

RNN (Recurrent Neural Network) dirancang untuk data sekuensial (teks, time series) — output tiap langkah waktu dipengaruhi input saat ini dan hidden state sebelumnya. RNN standar rentan vanishing/exploding gradient pada sekuens panjang.

LSTM (Long Short-Term Memory) mengatasi ini dengan cell state dan tiga gate:

  • Forget gate — memutuskan informasi lama mana yang dibuang.
  • Input gate — memutuskan informasi baru mana yang disimpan.
  • Output gate — memutuskan bagian cell state mana yang dijadikan output.

P10–11

Unsupervised Learning

Belajar dari data tanpa label — tujuannya menemukan struktur, pola, atau kelompok tersembunyi dalam data.

01K-Means Clustering

Mengelompokkan data ke dalam K cluster berdasarkan kedekatan jarak ke centroid.

  1. Tentukan jumlah cluster K, inisialisasi K centroid secara acak.
  2. Assign setiap titik data ke centroid terdekat.
  3. Update posisi centroid = rata-rata seluruh anggota cluster tersebut.
  4. Ulangi langkah 2–3 sampai centroid tidak berubah (konvergen).
Pemilihan K: gunakan Elbow Method (plot WCSS/inertia vs K, cari titik "siku") atau Silhouette Score (semakin mendekati 1, semakin baik pemisahan cluster).

02Hierarchical Clustering

Agglomerative (bottom-up): tiap titik mulai sebagai cluster sendiri, lalu digabung berulang hingga tersisa satu cluster besar. Divisive (top-down): kebalikannya, mulai dari satu cluster besar lalu dipecah. Hasilnya divisualisasikan sebagai dendrogram.

LinkageDefinisi Jarak Antar Cluster
SingleJarak minimum antar anggota dua cluster
CompleteJarak maksimum antar anggota dua cluster
AverageRata-rata jarak semua pasangan anggota
WardMinimalkan peningkatan variansi within-cluster

03DBSCAN (Density-Based Clustering)

Mengelompokkan berdasarkan kepadatan titik, tidak perlu menentukan jumlah cluster di awal, dan bisa mendeteksi noise/outlier.

  • Core point — punya ≥ minPts tetangga dalam radius ε (epsilon).
  • Border point — dalam radius ε dari core point, tapi tetangganya kurang dari minPts.
  • Noise point — bukan keduanya, dianggap outlier.
Kelebihan DBSCAN vs K-Means: bisa menemukan cluster berbentuk tidak beraturan dan tahan terhadap outlier, tapi sensitif terhadap pemilihan parameter ε dan minPts.

04Dimensionality Reduction: PCA & t-SNE

PCA (Principal Component Analysis) mencari sumbu (komponen utama) yang memaksimalkan variansi data, dihitung dari eigenvector matriks kovarians data. Digunakan untuk kompresi fitur & mengurangi noise.

t-SNE mereduksi dimensi tinggi ke 2D/3D khusus untuk visualisasi, mempertahankan kedekatan lokal antar titik — tidak cocok untuk preprocessing model prediktif karena non-deterministik dan berat secara komputasi.

05Association Rule Mining (Apriori)

Menemukan aturan asosiasi antar item, contoh klasik: analisis keranjang belanja ("market basket analysis").

Metrik AprioriSupport(A) = freq(A) / N Confidence(A→B) = Support(A∪B) / Support(A) Lift(A→B) = Confidence(A→B) / Support(B)

Lift > 1 berarti A dan B lebih sering muncul bersama dibanding jika independen (asosiasi positif).

P12

Instance-Based Learning

Disebut juga lazy learning — tidak membangun model eksplisit saat training, melainkan menyimpan seluruh data dan membandingkannya langsung saat prediksi.

01K-Nearest Neighbors (KNN)

Klasifikasi/regresi berdasarkan K tetangga terdekat dari titik data baru.

  • Klasifikasi: label = mayoritas (voting) label dari K tetangga terdekat.
  • Regresi: nilai prediksi = rata-rata nilai K tetangga terdekat.
  • Weighted KNN: tetangga yang lebih dekat diberi bobot lebih besar dalam voting.
Pemilihan K: K kecil → sensitif terhadap noise (overfitting); K besar → keputusan terlalu general (underfitting). K biasanya dipilih ganjil untuk menghindari seri pada klasifikasi biner.

03Lazy Learning vs Eager Learning

04Curse of Dimensionality

Saat jumlah fitur/dimensi bertambah banyak, jarak antar titik data menjadi semakin "seragam" sehingga konsep tetangga terdekat kehilangan makna — performa KNN dan metode berbasis jarak lainnya menurun. Solusi: reduksi dimensi (PCA) atau feature selection sebelum menerapkan KNN.

05Locally Weighted Regression

Variasi regresi yang membentuk model linear sederhana secara lokal di sekitar titik query, dengan bobot lebih besar untuk data training yang lebih dekat ke titik tersebut — menggabungkan sifat lazy learning dengan regresi.

P13

Hidden Markov Model (HMM)

Model probabilistik untuk data sekuensial, di mana state sebenarnya tersembunyi dan hanya observasi yang dapat dilihat.

01Komponen HMM

Sebuah HMM didefinisikan sebagai λ = (A, B, π):

  • State tersembunyi (hidden states) — kondisi yang tidak bisa diamati langsung.
  • Observasi (observations) — data yang benar-benar terlihat/terukur.
  • A — matriks transisi: probabilitas berpindah dari satu state ke state lain.
  • B — matriks emisi: probabilitas menghasilkan observasi tertentu dari suatu state.
  • π — distribusi awal: probabilitas state pada waktu awal.
Markov Assumption: state saat ini hanya bergantung pada state sebelumnya (bukan seluruh riwayat state).

02Tiga Masalah Utama HMM

MasalahPertanyaanAlgoritma
EvaluationBerapa P(O|λ)? Seberapa mungkin urutan observasi O muncul dari model λ?Forward Algorithm
DecodingUrutan state tersembunyi apa yang paling mungkin menghasilkan O?Viterbi Algorithm
LearningBagaimana estimasi parameter (A, B, π) terbaik dari data observasi?Baum-Welch Algorithm

03Detail Tiap Algoritma

  • Forward Algorithm — dynamic programming, menjumlahkan probabilitas semua kemungkinan jalur state secara efisien (dibanding brute-force yang eksponensial).
  • Viterbi Algorithm — mirip Forward tapi menggunakan operasi max bukan sum, untuk mencari satu jalur state paling mungkin (bukan total probabilitas semua jalur).
  • Baum-Welch Algorithm — kasus khusus algoritma Expectation-Maximization (EM); secara iteratif memperbarui A, B, π hingga konvergen tanpa membutuhkan label state yang benar.

04Contoh Aplikasi

Speech recognition (state = fonem tersembunyi, observasi = sinyal suara), Part-of-Speech tagging (state = kelas kata, observasi = kata), analisis urutan gen dalam bioinformatika.

P14

Reinforcement Learning

Agent belajar mengambil aksi terbaik melalui trial-and-error dengan berinteraksi pada suatu environment, dipandu oleh sinyal reward.

01Komponen Dasar

  • Agent — pengambil keputusan.
  • Environment — dunia tempat agent beroperasi.
  • State (S) — kondisi environment saat ini.
  • Action (A) — pilihan yang bisa diambil agent.
  • Reward (R) — umpan balik numerik atas suatu aksi.
  • Policy (π) — strategi/pemetaan dari state ke action.

02Markov Decision Process (MDP)

RL diformalkan sebagai MDP = (S, A, P, R, γ), dengan P = probabilitas transisi state, γ = discount factor (0–1) yang menentukan seberapa penting reward masa depan dibanding reward saat ini.

Return (total reward terdiskon)G_t = R_t+1 + γR_t+2 + γ²R_t+3 + ...

03Value Function & Bellman Equation

Value functionV(s) = E[G_t | S_t = s] Q-function (action-value)Q(s,a) = E[G_t | S_t = s, A_t = a] Persamaan BellmanV(s) = Σ_a π(a|s) Σ_s' P(s'|s,a)[R + γV(s')]

04Q-Learning vs SARSA

Q-Learning (off-policy)SARSA (on-policy)
UpdateQ(s,a) += α[R + γ·max Q(s',a') − Q(s,a)]Q(s,a) += α[R + γ·Q(s',a') − Q(s,a)]
Aksi berikutnyaSelalu aksi terbaik (greedy)Aksi yang benar-benar diambil policy
SifatBelajar dari kebijakan optimal, terlepas dari aksi yang dijalankanBelajar dari pengalaman aktual policy saat ini

05Eksplorasi vs Eksploitasi

Trade-off inti dalam RL: eksplorasi (mencoba aksi baru untuk menemukan info lebih baik) vs eksploitasi (memilih aksi terbaik yang sudah diketahui).

Epsilon-greedyDengan probabilitas ε → pilih aksi acak (eksplorasi) Dengan probabilitas 1−ε → pilih aksi terbaik saat ini (eksploitasi)

06Model-Based vs Model-Free

Model-based: agent punya/mempelajari model transisi P dan reward environment, lalu merencanakan aksi. Model-free (seperti Q-Learning, SARSA): agent belajar langsung dari pengalaman tanpa membangun model environment secara eksplisit.

P15

Ensemble Learning

Menggabungkan beberapa model (base learner) untuk menghasilkan prediksi yang lebih akurat dan stabil dibanding satu model tunggal.

01Bagging (Bootstrap Aggregating)

Melatih banyak model secara paralel, masing-masing pada subset data hasil bootstrap sampling (sampling with replacement). Hasil akhir digabung lewat voting (klasifikasi) atau rata-rata (regresi).

  • Contoh utama: Random Forest — bagging pada Decision Tree, ditambah pemilihan subset fitur acak di tiap split.
  • Tujuan utama: mengurangi variance (mencegah overfitting dari model yang kompleks).

02Boosting

Melatih model secara berurutan (sequential) — setiap model baru fokus memperbaiki kesalahan model sebelumnya, biasanya dengan memberi bobot lebih besar pada data yang salah diklasifikasi.

  • AdaBoost — menaikkan bobot sampel yang salah diprediksi di setiap iterasi.
  • Gradient Boosting — tiap model baru dilatih untuk memprediksi residual error (gradien loss) dari model sebelumnya.
  • XGBoost — implementasi gradient boosting yang dioptimalkan (regularisasi bawaan, lebih cepat & efisien).
  • Tujuan utama: mengurangi bias (memperbaiki model yang cenderung underfitting).

03Stacking

Menggabungkan prediksi dari beberapa base model berbeda (mis. KNN, SVM, Decision Tree) sebagai input fitur untuk model lapisan kedua (meta-learner) yang mempelajari cara terbaik mengombinasikan prediksi-prediksi tersebut.

04Voting Classifier

TipeCara Kerja
Hard VotingLabel akhir = kelas mayoritas dari semua model
Soft VotingLabel akhir = kelas dengan rata-rata probabilitas tertinggi antar model

05Bagging vs Boosting (Bias-Variance Tradeoff)

BaggingBoosting
Cara trainingParalel, independenSequential, tiap model bergantung sebelumnya
FokusMengurangi varianceMengurangi bias
Sensitif outlierTidak terlaluCukup sensitif (bobot naik pada kesalahan)
ContohRandom ForestAdaBoost, Gradient Boosting, XGBoost
Sering ditanya: kenapa Random Forest jarang overfitting meski terdiri dari banyak decision tree yang dalam? Karena tiap tree dilatih pada data & fitur berbeda (random), sehingga kesalahan masing-masing tree cenderung tidak berkorelasi dan saling menetralkan saat digabung (voting/rata-rata).

Tips Ujian

  • Hafalkan 3 masalah HMM beserta nama algoritmanya (Evaluation→Forward, Decoding→Viterbi, Learning→Baum-Welch) — ini paling sering muncul sebagai soal mencocokkan.
  • Pahami alasan di balik tiap teknik, bukan cuma nama — misalnya kenapa Dropout mencegah overfitting, atau kenapa Boosting mengurangi bias sementara Bagging mengurangi variance.
  • Latihan menghitung manual dengan angka kecil: satu iterasi K-Means, satu langkah update Q-Learning, atau menghitung Support/Confidence/Lift Apriori — soal hitungan sederhana sering keluar.
  • Buat tabel perbandingan sendiri di kertas coret-coretan sebelum ujian dimulai (mis. Bagging vs Boosting, Q-Learning vs SARSA) sebagai pemanasan mengingat.
  • Kalau soal berbentuk studi kasus, identifikasi dulu: ada label atau tidak? data sekuensial atau tidak? butuh reward/lingkungan interaktif atau tidak? — itu langsung mempersempit topik yang relevan.
